## TL;DR

NEO induces a latent compositional program τ (sequence of discrete primitives from a VQ-VAE codebook) from an observation pair (x, y), executes it through a shared transition model, selects the shortest accurate explanation via MDL, and (NEO-S) samples B candidate programs + majority-votes for test-time scaling. Evaluated on OTIB (GridWorld, Arithmetic Factorization, Image Editing) for compositional + length OOD generalization and transferability.

**Distilled for katgpt-rs (modelless, inference-time):** nothing not already shipped. NEO's six core mechanisms map onto shipped code as follows — (1) theory induction → Plan 290 PTG + MotifMiner + Plan 324 bisimulation + Plan 296 induced_cwm; (2) MDL length selection → Plan 215 MDL gate; (3) state grounding → Plan 312 viable_manifold_graph + `latent_functor/reestimation.rs`; (4) test-time scaling → PPoT + CLR + BoMSampler (all DEFAULT-ON, all beat majority vote); (5) transferability → Plan 290 TaR metric + `AnchorProfile`; (6) primitive discovery → Plan 290 MotifMiner + MotifAdmitter (exact analog). The paper's **value is its training recipe** (VQ-VAE codebook + transition operator + state-grounding loss, all learned end-to-end from raw observation via ELBO) → **riir-train**.

## 1. Paper Core Findings

### 1.1 The L2T paradigm

A *theory* is an executable compositional program τ = (z_i1, ..., z_iK) of length K over a finite set of primitive operations Z, executed as functional composition f_τ = f_z_iK ∘ ... ∘ f_z_i1. Given an observation pair (x, y), the model infers a latent program that explains the transition y = f_τ(x). Training data is i.i.d. phenomena {(x_n, y_n)} generated by unobserved programs; no language, task labels, or program supervision. Test-time requires **compositional generalization** (unseen program compositions within training length range) AND **length generalization** (programs longer than any seen in training).

### 1.2 NEO architecture (the VQ-VAE + ELBO core)

NEO maximizes p_θ(y|x) by introducing two latent variables: program τ = (z_i1, ..., z_iK) and execution trace s = (s_1, ..., s_{K+1}). Generative model: encoder p_θ(s_1|x), theory programmer p_θ(z_ik|s_k) (selects next primitive given current state), transition model p_θ(s_{k+1}|s_k, z_ik) (executes primitive). Variational posterior q_ϕ(τ, s|x, y) uses a goal-conditioned policy q_ϕ(z_ik|s_k, y) — the theory programmer steers the execution trace toward a latent state that explains y. The discrete primitive selection is realized via **VQ-VAE** (codebook E = {e_1, ..., e_M'}, commitment loss, Gumbel-softmax annealing). State grounding loss `L_state = Σ_k ‖s_k − sg[E_θ(D_θ(s_k))]‖²` anchors intermediate states to the encoder's latent manifold.

### 1.3 MDL-based adaptive program length

`k* = argmin_{k ∈ {1,...,K+1}} λ_MDL^k · ℓ(y, ŷ_k)` where λ_MDL > 1 penalizes longer programs exponentially. The model is updated by backpropagating loss only from the MDL-selected prediction ŷ_{k*}. Ablation: removing state grounding collapses training (primitiveness drops to 0.002); λ_MDL too large (1.2) forces over-short entangled programs; λ_MDL too small (0.8) allows over-decomposition.

### 1.4 Test-time scaling (NEO-S)

Sample B candidate theories from the probabilistic theory programmer, filter to those that reconstruct the support y^(1), majority-vote the most frequent program, then transfer the winner to the query input x^(2). Increasing B from 1 to 64 (GridWorld) or 1024 (Arithmetic) monotonically improves transferability; monolithic baselines remain flat. Sampling temperature adds a second axis.

### 1.5 Empirical results (OTIB)

- **GridWorld** (10×10, 4 motion primitives, train length 1–3, test length 4–8): NEO-S reaches 0.907–0.949 transfer at α=0.33 (hardest split). Monolithic baselines (Disc-Mono, Cont-Mono, Cont-Mono-Opt) collapse to ~0 on OOD transfer despite high self-explainability.
- **Arithmetic Factorization** (×2/×3/×5/×7, train length 1–3, test length 4–6): NEO-S reaches 0.696–0.799 length-OOD transfer. NEO alone (no search) gets ~0.02–0.04 on length OOD — the search is doing the heavy lifting for long horizons.
- **Image Editing** (8 primitives on CIFAR-10, train length 1–2, test length 3–4): NEO achieves lowest L1 across all α settings.

### 1.6 What the paper itself names as limitations

Small discrete primitive sets, short program lengths, controlled synthetic benchmarks, deterministic execution, reconstruction-based stopping. Primitive semantics induced only through reconstruction — not guaranteed to align with human-interpretable or causal concepts.

## 2. Distillation

### 2.1 The training recipe → riir-train (genuinely hard, genuinely training)

The paper's core contribution is **learning** the discrete primitive codebook E and the shared transition operator f_θ(s, z) from raw observation pairs via VQ-VAE commitment loss + state-grounding loss + ELBO. This is gradient descent through a discrete codebook — there is no modelless analog.

**§3.5 modelless unblock check (mandatory):**
- **Path 1 (freeze/thaw):** ❌ No frozen corrected snapshot can substitute — there is no base codebook to freeze; the codebook *itself* is what gets learned.
- **Path 2 (raw/lora reader-writer hot-swap):** ❌ A deterministically constructed LoRA cannot learn a discrete quantizer from continuous latents. The quantizer IS the learned object.
- **Path 3 (latent-space correction):** ⚠️ Partial. A pre-existing discrete vocabulary (`PrimitiveKind` enum + `PrimitiveKind::Composite(motif_hash)` from Plan 290) can serve as the codebook. The transition operators (`latent_functor`, `induced_cwm::InducedCwmKernel`, `evolve_hla`) can serve as f_θ. But this does not **discover novel primitive types from the observation stream** the way VQ-VAE training does — it composes/decomposes using a pre-existing + motif-grown vocabulary. Weaker than NEO's learned codebook.

**Conclusion:** The VQ-VAE codebook learning is a genuine riir-train dependency. The modelless path (use existing `PrimitiveKind` + composites as codebook) is available but strictly weaker — it cannot discover primitives that were never observed in any form. Document the dependency; route the training to riir-train.

### 2.3 Fusion — none novel (the prior-art surface is saturated)

NEO is the same research lineage as R264/P290. The three "genuinely additive modelless pieces" a naive reading would identify — (a) explicit `argmin_k` sweep over candidate program lengths, (b) decode→re-encode cycle-consistency as one primitive, (c) closed-loop induce-apply-measure-transfer — are each **small syntheses over already-shipped primitives**, not new capability classes:

- (a) k-sweep = a search loop wrapping Plan 215's `RegimeTransitionGate::evaluate()` at each k. Trivial wrapper, not a new primitive.
- (b) decode-encode consistency = compose Plan 312 `pullback_volume` (on-manifold check) with the existing transition operators. Pieces ship; the composition is mechanical.
- (c) closed-loop transfer = wire `induced_cwm::InducedCwmKernel` (induce) → execute on x2 → Plan 290 `TransferAsRecomposition::compute()` (measure). Pieces ship; the wiring is integration work, not research.

None of these individually or jointly produce a capability the existing stack cannot do. They are refinements of Plan 290's scope. Creating a new plan for them would duplicate Plan 290's charter.

### 2.4 Latent vs raw boundary (mandatory check)

Not applicable — no new boundary-crossing behavior. NEO's latent program induction operates entirely in latent space (s = E(x), s_y = E(y)); only the final reconstruction ŷ crosses to observation space. Our Plan 290 already enforces the same boundary discipline (PTG structure = raw/syncable for replay; motif embeddings = latent/local; TaR scalar = diagnostic). No change to the sync contract.

### Latent-space reframing check (mandatory per skill — primary framing)

- **HLA framing:** NEO = "given two HLA snapshots, infer the sequence of primitive affect-transitions that explains the delta". We have `evolve_hla` (forward evolution) but not "infer the primitive sequence from a snapshot pair". However, this is a search problem over the existing primitive vocabulary — not a new mechanism. The search would compose Plan 290's PTG (vocabulary) + Plan 215's MDL gate (cost) + existing transition operators. Mechanical, not novel.
- **Latent functor framing:** NEO's transition model f_θ(s, z) IS a latent functor application (apply operator z to latent state s). We ship rank-1 (`latent_functor`) and rank-k (FuncAttn) operators, BLAKE3-committed per `FunctorEntry`. The "theory" is a composition of functor applications. Composition ships; "infer the composition from a target state pair" is the search wrapper above.
- **CGSP framing:** NEO's theory programmer q_ϕ(z|s, y) is a goal-conditioned policy. CGSP is curiosity-driven (no fixed goal). The goal-conditioned variant is a specialization, not a new class.
- **Neuron-shard framing:** a NEO theory, once induced, could be frozen as a `KarcShard` or `ArchetypeBlendShard` (committed personality / forecaster Pod). The freeze/thaw envelope already covers this — no new artifact class needed.
- **LatCal framing:** no natural LatCal angle — NEO's transition operator is a learned neural map, not a deterministic committed fixed-point bridge.
- **DEC framing:** a NEO theory as a sequence of `exterior_derivative` / `codifferential` applications — the d∘d=0 identity gives program structure, but this is a stretch, not a natural reframing.

**No latent-space reframing yields a new capability.** Every reframing reduces to "search over the existing primitive vocabulary using existing operators, gated by the existing MDL gate". Adapter-routing framing would be even weaker.

### §3.6 defend-wrong PoC check

This PASS verdict is **architectural** (the mechanism ships under different vocabulary), NOT a **quality-parity** claim. We do NOT claim "our runtime matches NEO's transfer accuracy on OTIB" — we claim "the runtime analog of NEO's inference loop exists". Per §3.6, an architectural PASS requires grep + read (sufficient), NOT a PoC. The grep-and-read evidence (Plan 290 source + Plan 215 MDL gate + PPoT/CLR/BoMSampler benchmarks) confirms the mechanisms ship. If a future selling point ever claims quality parity with NEO on an OTIB-like benchmark, the PoC becomes mandatory at that point — tracked as a non-blocking note below.
