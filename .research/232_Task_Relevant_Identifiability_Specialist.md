## TL;DR

The paper proves two nonparametric identifiability theorems that hold *without any parametric or distributional assumption* and *without interventions*:

1. **Temporal task structure is identifiable** (Thm 1) — Tasks defined as **colliders across time steps** (`a_{t1} → g_i ← a_{t2}`) can be exactly recovered by simple **conditional independence (CI) tests** using a 4-element **band conditioning set** `Z_band(k,v,i) = {s_{kL−1}, s_{kL+1}, s_{vL−1}, s_{vL+1}} ∪ {g_i}`. Works with disconnected sequences, i.i.d., and arbitrarily interleaving tasks.

2. **Task-relevant latents are identifiable** (Thm 2) — Under a simple **sparsity regularization** `‖I(J_û)‖ ≤ ‖I(Ju)‖` on the task-latent Jacobian support, the estimated task-relevant latents are an invertible function of *only* the true task-relevant latents. **Without** sparsity, generalist models learn an **overcomplete** superset (Prop 2: `‖I(J_û)‖ ≥ ‖I(Ju)‖`).

3. **Algorithm 1** — O(N²) pairwise CI tests across segments recover the full task structure. No training. CI preserved under invertible observation map, so the test runs directly on observed data.

The conceptual move: **Generalist → Specialist is achieved by sparsity on the task-latent Jacobian, not by adding parameters.** This is an *inference-time* result. Tasks are *colliders* (coordinated actions targeting a shared plan), not confounders or mediators.

**Why this is bigger than LeJEPA (138) and OPD (231) for us:**

| | LeJEPA (138) | OPD (231) | This Paper (232) |
|---|---|---|---|
| Assumption | Latents must be Gaussian | None, observational | **None, nonparametric** |
| Object | Encoder weights | Parameter deltas | **Latent representation** |
| Identifiability | Linear (up to A) | None — observational only | **Per-task subgroup (up to h_k)** |
| Operates on | Training | Training-side observation | **Inference-time, latent + temporal** |
| Test | Hermite spectrum | Off-principal projection | **Band conditioning CI test** |
| Use case | Diagnostic | Adapter storage / retrieval | **KV selection + sparse projection** |

This research closes the theoretical gap that caused MSA sparse attention (Plan 256, Research 225) to **GOAT-FAIL** — that sparsity was heuristic. With Theorem 2, sparsity is *provably* the minimal sufficient representation. We can re-derive sparse attention through identifiability.

## Paper Core

### Setting

Observed `{o_t}` generated by latent `{s_t}` with actions `{a_t}`:

```
o_t = f_t(s_t)            // f_t diffeomorphism onto image (fully general, nonparametric)
s_{t+1} = F_t(s_t, a_t, ξ_t)    if t→t+1 connected
        = F'_t(ξ_t)               if disconnected (allows i.i.d.!)
a_t = π_t(s_t, η_t)
```

**Task** `g_i` = **collider** across time steps: `s_t → a_t → g_i ← a_{t'} ← s_{t'}`.

### Why Tasks Are Colliders (Remark 1)

- Confounder/mediator (`a_{t1} ← g_i → a_{t2}` or `a_{t1} → g_i → a_{t2}`) implies `{s_t1, a_t1} ⊥ {s_t2, a_t2} | g_i` — treats plan steps as isolated.
- **Collider** (`a_{t1} → g_i ← a_{t2}`) implies `{s_t1, a_t1} ⊭ {s_t2, a_t2} | g_i` — captures coordinated planning.

This is the formal reason the structure is recoverable: collider-conditional *dependence* is the signal.

### Theorem 1 (Temporal Task Structure Identifiability)

```
g_i is relevant to segments S_k and S_v  ⟺  s_{kL} ⊭ s_{vL} | Z_band(k,v,i)
```

where `Z_band(k,v,i) = {s_{kL−1}, s_{kL+1}, s_{vL−1}, s_{vL+1}} ∩ {s_1..s_T} ∪ {g_i}`.

**Proof intuition:** Band conditioning blocks all purely-temporal d-connecting paths (via local Markov blanket). Any remaining dependence *must* route through a task collider. Since only `g_i` is in `Z_band`, all other tasks are closed colliders and block. The four admissible d-connecting path forms are enumerated (Lemma 2).

**Corollary 1:** Test works for *any* `s_j ∈ S_k, s_q ∈ S_v` (not just boundaries) — segment homogeneity.

### Algorithm 1 (Global Task Structure Discovery)

```
Input:  Segments S_1..S_N (length L ≥ 2); tasks g_1..g_M
Output: Per-segment and per-step task labels

For each task g_i:
    For each segment pair (k, v):
        If s_{kL} ⊭ s_{vL} | Z_band(k, v, i):   # CI test
            Add g_i to T(S_k) and T(S_v)
Per-step: T(t) := T(S_k) where t ∈ S_k
```

**Complexity:** O(M · N²) CI tests. Scales with horizon T, *not* observation dim d_o. Runtime Table 2: T=20, M=4 → 0.02s.

### Proposition 2 (Generalist = Overcomplete)

Without sparsity, for any task `g_k`:

```
‖I(J_û)_{k,·}‖ ≥ ‖I(Ju)_{k,·}‖
```

The estimated task-relevant latent support is a **superset** of the true one. Generalist models over-complete — they encode all dependencies but fail to isolate minimal sufficient ones.

### Theorem 2 (Specialist via Sparsity)

With sparsity regularization `‖I(J_û)‖ ≤ ‖I(Ju)‖`:

```
ŝ_{t,π(I_k)} = h_k(s_{t,I_k})     // invertible function of true task-relevant latents ONLY
```

The estimated task-relevant latents disentangle from irrelevant ones, up to an invertible map `h_k`. **This is the formal definition of "specialist".**

### Empirical Validation

- **Synthetic** (Fig 3): Dominates CCA, Group Lasso, SelTask on accuracy + MCC for T ∈ [8,20], M ∈ [2,10].
- **Real-world** (Fig 4, SportsHHI): mAP 0.25 vs Leap 0.12, Base 0.09, Slowfast 0.11, VitB 0.12.
- **Specialist representation** (Fig 5): R² for relevant parts high, for irrelevant parts low — clean disentanglement.
- **Controllable generation** (Fig 6/7): With sparsity, only intended concepts modified. Without, irrelevant factors (color, identity) leak.

### CMI Surrogate (Appendix C)

For high-dim settings, replace CI test with conditional mutual information estimated via InfoNCE:

```
I(s_{kL}; s_{vL} | c_i)  where  c_i = h_φ(Z_band(k,v,i))
L_NCE = E[log exp f_θ(s_{kL}, s_{vL}, c_i) / (exp f_θ(...) + Σ_j exp f_θ(s_{kL}, s̃_{vL}^{(j)}, c_i))]
```

This is the inference-time implementable form.

### Three Fusions (All Modelless, All Public)

#### Fusion A: Band-Conditioned KV Segment Selector (BCKVSS)

**Core idea:** Treat KV cache tokens as time steps. Treat "task" = current query's semantic intent (the collider). For each candidate past segment, run the band-conditioning CI test:

```
retain(S_v in KV)  ⟺  s_{kL} ⊭ s_{vL} | Z_band(k, v, q)
```

where `q` is the current query embedding (treated as `g_i`).

**Why it's a generalization of retrieval-head sparse attention:** Retrieval heads are *learned* heuristics for "which tokens matter." Band conditioning is a *provable* test — the retained tokens are exactly those that share a task-collider with the query, by Theorem 1.

**Why it's DRY:** Replaces three things with one primitive:
1. `rt_turbo` retrieval-head selection (heuristic).
2. `StillKV` perceiver compaction (learned).
3. `octopus` octahedral compression (geometric).

All collapse into "band-conditioning CI test against current query."

**GOAT targets:**
- G1: At least 2× reduction in CI test calls vs naive O(N²) (use segment length L ≥ 2 + Corollary 1).
- G2: Selection MCC ≥ 0.85 on synthetic SCM test (paper Fig 3 level).
- G3: KV cache size reduction ≥ 40% with no quality drop on long-context benchmark.

**Integration point:** New file `src/band_conditioner.rs` + trait extension on `ConstraintPruner` (or new `SegmentSelector` trait to keep SRP).

#### Fusion B: Specialist Latent Projection (SPLAT)

**Core idea:** Inference-time projection of hidden states onto the minimal task-relevant subspace, gated by sparsity on the Jacobian support (Theorem 2).

```
h_specialist = h · M_sparse
where M_sparse[i,j] = 1 iff j ∈ I(J_û)_{i,·}   # the discovered support
```

The mask `M_sparse` is computed once per (query, adapter) pair via:
1. Estimate Jacobian support from a small sample of activations (Prop 2's `N_i` set — paper says `|I(Ju)|` samples suffice).
2. Apply sparsity bound: keep only the minimum-coordinates whose removal does not violate `‖I(J_û)‖ ≤ ‖I(Ju)‖`.

**Why it unblocks Plan 264:** Plan 264 Phase 2 (Off-Principal Retrieval) needs an inference-time consumer of `SparseTaskVector`. SPLAT is that consumer — it applies the sparse mask to hidden states at inference, exactly as Theorem 2 prescribes.

**Why it's the MSA rescue:** MSA sparse attention (Plan 256) GOAT-FAILED because the sparsity mask was learned without theory. SPLAT provides a theory-backed mask: the support of the task-latent Jacobian. We can re-derive sparse attention through SPLAT.

**GOAT targets:**
- G4: Specialist projection reduces hidden-state dim by ≥ 30% with quality parity (paper R² gap, Fig 5).
- G5: Mask discovery cost ≤ `|I(Ju)|` samples (paper Prop 2's `N_i` cardinality).
- G6: MSA rescue — SPLAT-masked attention matches dense attention quality at 50% density (where MSA failed).

**Integration point:** New file `src/specialist_projection.rs`. Consumes `SparseTaskVector` from Plan 264. Feature gate `specialist_projection` (depends on `sparse_task_vector`).

#### Fusion C: Collider-Consistency ConstraintPruner (CCCP)

**Core idea:** A new `ConstraintPruner` impl that rejects DDTree branches whose action sequence breaks collider consistency. A "task" in DDTree = a partial plan. The pruner checks: does extending this branch preserve the collider structure?

```
is_valid(depth, token_idx, parent_tokens) :=
    ∃ task g_i such that for all completed segments in parent_tokens,
    s_{kL} ⊭ s_{vL} | Z_band(k, v, i)   # the collider signal is preserved
```

If no such `g_i` exists, the branch is dead — it cannot be part of any coherent plan.

**Why it's the first theory-backed pruner:** All our existing pruners (`ConstraintPruner` impls) are either rule-based (game constraints) or learned (bandit). CCCP is the first with an identifiability theorem behind it.

**Why it's composable:** Implements the existing `ConstraintPruner` trait. Zero-cost when disabled (returns `true`). Can be combined with any other pruner via existing composition patterns.

**GOAT targets:**
- G7: Pruner rejects ≥ 90% of structurally-dead branches on synthetic interleaved-task benchmark.
- G8: Combined with bandit, improves DDTree efficiency by ≥ 25% (fewer expansions to find goal).
- G9: Zero overhead when no tasks are tracked (early return).

**Integration point:** New file `src/collider_pruner.rs`. Implements `ConstraintPruner` trait. Feature gate `collider_consistency`.

## Open Questions for Implementation

1. **CMI critic warm-start:** Paper's InfoNCE critic needs training. For pure modelless, can we use a frozen pretrained critic (e.g., existing `MaxSim` head) as `f_θ`? — Likely yes, treat as a fixed feature extractor.
2. **Segment length choice:** Paper says L=2 is sufficient. For KV cache, what's the natural unit? Token-level L=2 is too fine; chunk-level (e.g., 32 tokens) might match semantic boundaries better. — Phase 1 will sweep L ∈ {2, 8, 32, 128}.
3. **Jacobian support estimation cost:** Paper's Prop 2 needs `|I(Ju)|` samples. For inference-time SPLAT, can we approximate via finite differences on a single forward pass? — Yes, with one extra backward-free Jacobian-vector product.
