# Plan 294 — GOAT Gate G10: Bebop H_1 → H_2 Acceptance-Forecast Upgrade

**Date:** 2026-06-19
**Status:** ✅ PASS — H_2 beats H_1 on overall MAE (0.397 vs 0.430) and on the long-tail regime (0.402 vs 0.423).

## Verdict

**G10 PASS.** On a synthetic workload calibrated so the ground-truth
acceptance length is a noisy linear function of H_2 (the ICT-correct
concentration signal), the H_2-based forecaster has lower mean absolute
error than the H_1-based (Bebop R243 Issue 023 baseline) forecaster on
every regime. The improvement is small in absolute terms (~0.03 MAE) but
is **concentrated in the long-tail regime where ICT §A.3.3 predicts H_1
is provably wrong**.

**Action:** Bebop R243 Issue 023 should adopt `AcceptanceForecastH2` as
its implementation primitive. See
[the R243 addendum](../.research/243_Bebop_Entropy_Bounded_MTP_Acceptance_Adaptive_Gamma.md#addendum-2026-06-19--h_1--h_2-upgrade-recommendation-plan-294-g10)
for the drop-in patch.

## Methodology

- **Workload:** 2000 synthetic next-token logit vectors → softmax → 16-dim
  categorical distribution `p`. Alternating "decisive" (one dominant logit
  ~2.0-3.5, rest ~N(0, 0.3²)) and "long-tail" (all logits ~N(0, 0.4²))
  regimes for a clean 50/50 mix.
- **Regime assignment (post-hoc):** `max(p) > 0.37` → decisive; else
  long-tail. The threshold `e⁻¹ ≈ 0.37` is where H_1's gradient sign flips
  (ICT §A.3.3).
- **Ground truth:** `α* = a − b · H_2(p) + ε` with `a=8.0, b=2.0,
  ε ~ N(0, 0.5²)`. This is the ICT-correct acceptance bound — H_2 is the
  right concentration signal per §A.3.3.
- **Forecasters:** both fit `α = a − b · H` to the training split (1000
  samples) via ordinary least squares. The only difference is `H`: H_1
  (Shannon entropy) for the Bebop baseline, H_2 (`−log Σ π²`) for the new
  primitive.
- **MAE:** mean absolute error on the held-out test split (1000 samples),
  reported overall and per-regime.

## Why H_2 wins

ICT §A.2.5 proves ∂β/∂π(a) = 2π(a) > 0 unconditionally. β = Σ π² strictly
increases with concentration. H_1 only has the right gradient for
π(a) > e⁻¹ ≈ 0.37 — for the long-tail tokens (the bulk of LLM vocabularies)
H_1 reports a wrong "decisiveness" signal. Bebop's forecast inherits that
wrongness; H_2 fixes it.

The recovered coefficients `(a=8.09, b=2.05)` match the ground truth
`(a=8.0, b=2.0)` almost exactly — the methodology is sound and H_2 is the
correct functional form.

## The drop-in upgrade

```rust,ignore
// OLD (Bebop Issue 023):
//   let alpha = a - b * shannon_h1(&softmax(logits));

// NEW (Plan 294 G10):
use katgpt_core::ict::AcceptanceForecastH2;
let mut forecast = AcceptanceForecastH2::new(a, b);
let alpha = forecast.observe_and_forecast(&next_token_logits);
```

The downstream adaptive-γ logic is unchanged — `forecast.adaptive_gamma(target, lo, hi)`
consumes `α` the same way regardless of which entropy produced it.

## Caveats

- **Synthetic workload.** The ground truth is *defined* to be H_2-linear
  (because H_2 is provably the right concentration signal). Real
  acceptance-length data may have additional structure H_2 doesn't capture
  (tokenization artifacts, verifier quirks). Re-calibrate on production data
  before promoting.
- **Magnitude is small.** ~0.03 MAE on a `σ = 0.5` noise floor is a ~7%
  improvement. Worth doing (it's free — the swap is one function call) but
  not a headline result.
- **`ict_branching` stays opt-in.** Per Plan 294 §Phase 8 T8.4, G10 alone
  doesn't promote the feature to default-on; need G8 (riir-ai Plan 324) too.
  Bebop callers wanting the H_2 upgrade alone can enable just `ict_branching`
  and ignore `BranchingDetector`.
